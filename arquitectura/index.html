    Arquitectura del sistema¶

    Diagrama general¶

    El stack completo corre en la red interna backend. Todos los contenedores se comunican usando el nombre del servicio como hostname.

    Flujo de un experimento¶

    sequenceDiagram
    participant J as JupyterLab
    participant P as mlflow-proxy
    participant M as MLflow
    participant S as MinIO

    J->>+P: mlflow.start_run()
    P->>+M: POST /api/2.0/mlflow/runs/create<br/>(Host: localhost)
    M-->>-P: run_id
    P-->>-J: run_id

    J->>J: model.fit(X_train, y_train)
    J->>+P: mlflow.log_params / log_metrics
    P->>M: forward
    M-->>-J: OK

    J->>+P: mlflow.sklearn.log_model(model)
    P->>M: registrar artefacto
    M->>+S: PUT s3://mlflow/<run_id>/model/
    S-->>-M: OK
    M-->>-P: artifact_uri
    P-->>-J: OK

    Flujo de una predicción¶

    sequenceDiagram
    participant C as Cliente
    participant A as FastAPI
    participant P as mlflow-proxy
    participant M as MLflow
    participant S as MinIO

    Note over A: Al arrancar el contenedor
    A->>P: load_model("models:/modelo/Production")
    P->>M: GET model metadata
    M->>S: GET artefactos del modelo
    S-->>A: modelo deserializado

    Note over C,A: En cada request
    C->>+A: POST /predict {features}
    A->>A: predict(features)
    A-->>-C: {prediction, probability}

    Flujo de reentrenamiento (Airflow DAG)¶

    flowchart TD
    SCHED([⏱️ Schedule\nCeleryExecutor]) --> LOAD[Cargar dataset\nCSV desde disco]
    LOAD --> PREP[Preprocesamiento\nColumnTransformer]
    PREP --> OPT[Búsqueda Optuna\nhiperparámetros]
    OPT --> TRAIN[Entrenamiento\n+ log MLflow run]
    TRAIN --> EVAL{¿Mejor que\nmodelo en prod?}
    EVAL -->|Sí| REG[Registrar en\nMLflow Model Registry]
    EVAL -->|No| END([Fin · sin cambios])
    REG --> PROM[Promover a\nProduction stage]
    PROM --> RELOAD[FastAPI recarga\nmodelo automáticamente]
