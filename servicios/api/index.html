    FastAPI — Serving de predicciones¶

    La API carga el modelo desde el MLflow Model Registry al arrancar y expone predicciones en tiempo real. Siempre usa la última versión registrada de airline-satisfaction-best.

    Acceso¶

    Interfaz URL
    Swagger UI http://localhost:8000/docs
    ReDoc http://localhost:8000/redoc

    Endpoints¶

    Método Path Descripción
    GET / Info de la API (nombre, versión)
    GET /health Estado del servicio y si el modelo está cargado
    POST /reload Recarga el modelo desde MLflow sin reiniciar el contenedor
    POST /predict Predice la satisfacción de un pasajero

    GET /health¶

    { "status": "ok", "model_loaded": true }

    503 antes de entrenar

    Si el DAG todavía no corrió y no hay modelo registrado, /predict retorna 503: 

    { "detail": "Modelo no disponible. Ejecutar el DAG de entrenamiento antes." }

    POST /predict¶

    Features de entrada¶

    El request requiere 22 campos que describen al pasajero y su experiencia de vuelo.

    Campo Descripción Ejemplo
    gender Género "Male" / "Female"
    customer_type Tipo de cliente "Loyal Customer" / "disloyal Customer"
    type_of_travel Propósito del viaje "Business travel" / "Personal Travel"
    travel_class Clase del ticket "Business" / "Eco" / "Eco Plus"
    Campo Tipo Descripción
    age int (0–120) Edad del pasajero
    flight_distance int ≥ 0 Distancia del vuelo en millas
    departure_delay_in_minutes int ≥ 0 Retraso de salida
    arrival_delay_in_minutes float ≥ 0 Retraso de llegada

    Catorce ratings de servicio, todos enteros de 0 a 5:

    Campo Descripción
    inflight_wifi_service WiFi a bordo
    departure_arrival_time_convenient Conveniencia del horario
    ease_of_online_booking Reserva online
    gate_location Ubicación de la puerta
    food_and_drink Comida y bebida
    online_boarding Embarque online
    seat_comfort Confort del asiento
    inflight_entertainment Entretenimiento
    on_board_service Servicio a bordo
    leg_room_service Espacio para piernas
    baggage_handling Manejo de equipaje
    checkin_service Check-in
    inflight_service Servicio en vuelo
    cleanliness Limpieza

    Ejemplo de request¶

    curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "gender": "Male",
    "customer_type": "Loyal Customer",
    "age": 35,
    "type_of_travel": "Business travel",
    "travel_class": "Business",
    "flight_distance": 1500,
    "inflight_wifi_service": 4,
    "departure_arrival_time_convenient": 4,
    "ease_of_online_booking": 3,
    "gate_location": 3,
    "food_and_drink": 4,
    "online_boarding": 5,
    "seat_comfort": 4,
    "inflight_entertainment": 5,
    "on_board_service": 4,
    "leg_room_service": 4,
    "baggage_handling": 5,
    "checkin_service": 4,
    "inflight_service": 4,
    "cleanliness": 4,
    "departure_delay_in_minutes": 0,
    "arrival_delay_in_minutes": 0.0
  }'

    Response 200 OK¶

    {
  "prediction": "satisfied",
  "probability_satisfied": 0.9231,
  "model_version": "3"
}
    Campo Descripción
    prediction "satisfied" o "neutral or dissatisfied"
    probability_satisfied Probabilidad de la clase positiva (0–1), calculada con predict_proba si el modelo lo soporta
    model_version Versión en el MLflow Registry

    POST /reload¶

    Recarga el modelo desde Registry sin reiniciar el contenedor. Llamarlo después de que Airflow registre una nueva versión.

    curl -X POST http://localhost:8000/reload
# → {"status": "model reloaded", "model_loaded": true}

    Lógica de carga del modelo¶

    api/main.py — _load_model()
    client = mlflow.MlflowClient()
versions = client.search_model_versions(f"name='{REGISTERED_MODEL_NAME}'")
latest = sorted(versions, key=lambda v: int(v.version))[-1]  # (1)

model = mlflow.sklearn.load_model(f"runs:/{latest.run_id}/model")

local_path = client.download_artifacts(latest.run_id, "feature_columns.json")  # (2)
with open(local_path) as f:
    feature_columns = json.load(f)
    1. Toma la versión con el número más alto, no necesariamente la que está en stage Production.
    2. feature_columns.json se guarda como artefacto en cada run de Airflow. Garantiza que el preprocessing use exactamente las mismas columnas con las que se entrenó.

    Workflow completo

    Airflow DAG → MLflow Registry (nueva versión)
                                    ↓
                      POST /reload → API usa el nuevo modelo
