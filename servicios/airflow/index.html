    Airflow — Orquestación¶

    Arquitectura (CeleryExecutor)¶

    Airflow usa CeleryExecutor con Redis/Valkey como broker. Las tasks se distribuyen a workers y los resultados se persisten en PostgreSQL.

    graph LR
    SCH[scheduler] -->|publica tasks| REDIS[(Redis)]
    REDIS -->|consume| WRK[worker]
    SCH --> PG[(PostgreSQL\nairflow_db)]
    API[apiserver\n:8080] --> PG
    WRK --> PG

    DAG: airline_satisfaction_training¶

    Archivo: airflow/dags/airline_satisfaction_dag.py

    Entrena 4 modelos en paralelo con búsqueda de hiperparámetros (Optuna), y registra el mejor en el MLflow Model Registry.

    Configuración del DAG¶

    Parámetro Valor
    schedule None (manual o via API)
    catchup False
    retries 1 (delay: 5 min)
    dagrun_timeout 120 minutos
    N_TRIALS 10 (trials de Optuna por modelo)
    Tags ml, training, airline-satisfaction

    Grafo de tareas¶

    flowchart TD
    LOAD["📂 load_and_prepare_data\nCarga CSVs, preprocesa\nguarda Parquet en /tmp"] --> LR
    LOAD --> KNN
    LOAD --> RF
    LOAD --> XGB

    LR["📊 train_logistic_regression\nLogisticRegressionCV\ncv=10, max_iter=1000"]
    KNN["📊 train_knn\nKNeighborsClassifier\n+ Optuna"]
    RF["📊 train_random_forest\nRandomForestClassifier\n+ Optuna"]
    XGB["📊 train_xgboost\nXGBClassifier\n+ Optuna + early stopping"]

    LR --> SEL
    KNN --> SEL
    RF --> SEL
    XGB --> SEL

    SEL["🏆 select_and_register_best\nSelecciona por F1 score\nregistra en MLflow Registry"]

    Los 4 tasks de entrenamiento corren en paralelo (CeleryExecutor distribuye a workers disponibles).

    Detalle de cada task¶

    Carga los CSVs de datasets/aerolineas/, aplica el pipeline de preprocesamiento de src/data_loader.py y serializa los splits como Parquet en /tmp para que los tasks de entrenamiento los lean sin re-procesar.

    Salida: {"data_dir": "/tmp/airline_sat_<uuid>/"}
Archivos: X_train.parquet, y_train.parquet, X_test.parquet, y_test.parquet

    Imprime en los logs el shape de cada split: 

    Train: X=(103904, N), y=(103904,)
Test:  X=(25976, N),  y=(25976,)

    LogisticRegressionCV con cv=10 y max_iter=1000. Usa mlflow.sklearn.autolog() para loguear automáticamente parámetros y el modelo. No usa Optuna (el modelo ya hace CV interno).

    Incluye el preprocessor (ColumnTransformer) dentro de un Pipeline de sklearn.

    Búsqueda con Optuna (n_trials=10). Hiperparámetros buscados:

    Parámetro Rango
    n_neighbors 1–50
    weights uniform / distance
    p 1–2 (Manhattan / Euclidean)

    KNN sí requiere scaling → aplica el preprocessor antes de entrenar y guarda las features transformadas.

    Cada trial se loguea como un run anidado en MLflow (visible en la UI bajo el run padre KNN).

    Búsqueda con Optuna (n_trials=10). Hiperparámetros buscados:

    Parámetro Rango
    n_estimators 50–400
    criterion gini / entropy
    max_depth 3–30
    min_samples_split 2–20
    min_samples_leaf 1–20

    Random Forest no requiere scaling → se entrena directamente sobre las features originales.

    Búsqueda con Optuna (n_trials=10) + MedianPruner (desecha trials malos temprano). Usa early stopping con un split interno 80/20. Hiperparámetros buscados:

    Parámetro Rango
    n_estimators 50–400
    max_depth 2–10
    grow_policy depthwise / lossguide
    learning_rate 1e-4 – 1.0 (log)
    gamma 1e-8 – 1.0 (log)
    min_child_weight 1–10
    subsample 0.5–1.0
    colsample_bytree 0.5–1.0
    reg_alpha 1e-8 – 1.0 (log)
    reg_lambda 1e-8 – 1.0 (log)

    Recibe los resultados de los 4 modelos, los ordena por f1_score decreciente e imprime la tabla comparativa en los logs:

    Modelo                    F1
-----------------------------------
xgboost                   0.9543
random_forest             0.9421
logistic_regression       0.8876
knn                       0.9102

    Llama a register_best_model() → registra el run ganador en airline-satisfaction-best y devuelve el número de versión.

    Triggerear el DAG¶

    1. Ir a DAGs → airline_satisfaction_training
    2. Click en ▶ (Trigger DAG)
    docker exec airflow_apiserver \
  airflow dags trigger airline_satisfaction_training

    curl -X POST http://localhost:8080/api/v2/dags/airline_satisfaction_training/dagRuns \
  -H "Content-Type: application/json" \
  -u airflow:airflow \
  -d '{}'

    Acceso a la UI¶

    http://localhost:8080
Usuario: airflow / Password: airflow
